# 79. ユーザーテストのシナリオ作成

## 1. はじめに
アプリができたら、リリース前に「テスト」が必要です。
しかし、「適当にいじってみて」と頼んでも、人は普段通りの操作しかしません。
バグを見つけるための**「テストシナリオ（手順書）」**をGeminiに作らせましょう。

![ユーザーテストのシナリオ](./../attachments/04_79_api_connection_1769361933354.png)

## 2. シナリオ生成
> **プロンプト例**:
> 「在庫管理アプリの受入テストを行います。
> 機能の不具合を見つけるために、**正常な操作だけでなく、イレギュラーな操作**も含めたテストシナリオを5つ作ってください」

**Geminiの提案例**:
1.  **正常系**: 商品を登録し、在庫数が正しく反映されるか。
2.  **異常系**: 在庫数に「マイナス」や「文字」を入力してエラーになるか。
3.  **負荷**: 写真を連続で10枚登録しても落ちないか。
4.  **オフライン**: 機内モードで入力し、通信復帰後に同期されるか。

## 3. テスト実施
このシナリオをテスター（同僚など）に渡し、「この通りに意地悪な操作をしてみて」と頼めば、リリース前に致命的なバグを潰せます。

---

## 4. まとめ
「自分は間違えない」という思い込みが最大のバグです。
AIという冷静な第三者に、「壊し方（テスト方法）」を教えてもらい、頑丈なアプリに仕上げましょう。
