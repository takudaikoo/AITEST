# 72. 問い合わせフォームの自動振り分けロジック構築

## 1. はじめに
「問い合わせが全部『代表アドレス』に来て、担当者に転送するのが大変…」
問い合わせの内容に応じて、自動的に担当者（営業部、技術部、総務部など）に振り分けられたら楽ですよね。
GASを使ったこのロジックも、Geminiとなら作れます。

![問い合わせの自動振り分け](./../attachments/04_72_ocr_digitization_1769361818826.png)

## 2. ロジック生成プロンプト
Googleフォームと紐付いたスプレッドシートを用意します。

> **プロンプト例**:
> 「フォームの回答（C列：問い合わせ種別）に応じて、通知メールの宛先を変えるGASコードを書いて。
> *   『製品について』なら -> `sales@company.com`
> *   『不具合報告』なら -> `support@company.com`
> *   『その他』なら -> `info@company.com`
>
> メールの件名は『【種別】お名前様より』としてください」

## 3. キーワード判定
選択肢だけでなく、「本文（D列）に『急ぎ/緊急』という言葉が含まれていたら、件名に【至急】とつけ足して」といった**キーワード判定**のロジックも追加できます。

## 4. まとめ
「メールを見て転送する」という人間がやるにはあまりに単純な作業こそ、自動化の最優先ターゲットです。
スピード対応は顧客満足度にも繋がります。
